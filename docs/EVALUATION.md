# Evaluation approach and results

## What we measure

1. **Grounding / hallucination control**  
   - **Extractive mode:** every bullet must originate from a retrieved chunk; citations are mechanically aligned.  
   - **LLM mode:** prompt requires `[E#]` citations on factual lines; post-hoc checks can verify tags exist in `evidence`.

2. **Retrieval quality (offline)**  
   - Hybrid vs dense-only: for synthetic queries containing rare tokens (e.g., `$1,850.00`), BM25 should rank the rent clause higher than dense-only when embeddings underweight exact numerics.  
   - **RRF** merges lists without score calibration—cheap and robust.

3. **Edit loop signal**  
   - After submitting an operator correction, `retrieve_preferences` for the same `document_id` should return the stored “FROM/TO” snippet text (manual spot-check via Chroma query or `/drafts/generate` with logging `retrieval_debug`).

## How to rerun locally

```powershell
cd legal_workflow
.\.venv\Scripts\activate
python -m pytest -q
```

Use `samples/input/synthetic_lease.pdf` (generated by `scripts/build_sample_pdf.py`) as a controlled document. Upload via `/documents/upload`, generate a draft, then POST `/edits/feedback` with a deliberately rephrased final draft; regenerate and compare wording drift toward operator preferences.
